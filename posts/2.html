发表于2022-07-20|更新于2022-07-26

- 使用软编码的好处

硬编码: 将数据直接写入代码进行编译开发,若发生需要更改的问题,就需要更改源代码
软编码: 将数据与源代码解耦(分开处理), mybatis将sql于底层代码分离, 后期维护修改更为方便

- mybatis入门使用

流程：
- 定义pojo类(xxxx.pojo.类名)
- 加载核心配置文件,获取SqlSessionFactory对象
- 获取SqlSession对象,执行sql语句
- 释放资源

- Mapper代理

目的: 解决原生方式中的硬编码(入门代码第3步); 简化后期执行sql
步骤:

  1. 定义与sql映射文件同名的Mapper接口,并将Mapper接口与sql映射文件置于同一目录下
  2. 设置sql映射文件的namespace属性为Mapper接口全限定名
  3. 在Mapper接口中定义方法,方法名就是sql映射文件中sql语句的id,并保持参数类型和返回值类型一致
  4. 编码：通过SqlSession的getMapper方法获取Mapper接口的代理对象; 调用对应方法完成sql的执行

包扫描: 若Mapper接口名称与SQL映射文件名称相同,且在同一目录下,可使用包扫描的方式简化sql映射文件加载,即同一个包的文件只需要于mybatis-config.xml中输入以下代码,就无需在一条条加载映射文件: <package name="包路径"/>

eg:用Mapper代理实现查询全部数据操作

  1. mapper接口书写:List<students> selectAll();
  2. xml处书写: //1. 加载mybatis核心配置文件，获取SqlSessionFactory
    String resource =”mybatis-config.xml”;
    inputStream inputStream = Resources.getResourceAsStream(resource);
    SqlSessionFactory sqlSessionFactory =new SqlSessionFactoryBuilder().build(inputStream);

//2. 获取SqlSession对象，用它来执行sql
SqlSession sqlSession =sqlSessionFactory.openSession();

//3. 获取Mapper接口的代理对象并执行
testDemoMapper testDemoMapper =sqlSession.getMapper(testDemoMapper.class);

//4.执行方法
List students =testDemoMapper.selectAll();
System.out.println(students);

//5. 释放资源
sqlSession.close();

  select * from studentsinformation

        <if test="statement !=null">

        </if>
        <if test="nums !=null">
            and nums =#{nums}
        </if>
        <if test="name != null and name !='' ">
            and name =#{name}
        </if>
    </where>
</select>

<select id="selectByConditionSinger" resultType="myTest.pojo.students">
    select * from studentsinformation
    <where>
        <choose>
            <when test="nums !=null">
                nums =#{nums}
            </when>
            <when test="name !=null and name!='' ">
                name =#{name}
            </when>
            <when test="statement !=null">
                statement =#{statement}
            </when>
        </choose>
    </where>
</select>

    insert into studentsinformation(name, sex, birth, score, email, phone, statement)
